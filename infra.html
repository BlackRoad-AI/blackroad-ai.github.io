Route Everything to Your Infra

Use Tailscale for zero-config encrypted mesh networking and Cloudflare Tunnel for public HTTPS access — no open firewall ports, no external providers in the data path.

Architecture Overview

The goal: every BlackRoad AI service request terminates on your hardware, not on a cloud provider's servers.

Part 1 — Tailscale Mesh

Tailscale creates a WireGuard-based encrypted overlay network between all your devices. Once set up, every Pi and server can reach every other node via a stable 100.x.x.x address or via MagicDNS hostnames, regardless of physical location.

Install Tailscale on each Pi / server

# Run this on each node
curl -fsSL https://tailscale.com/install.sh | sh

# Bring the node up (will open a browser auth URL)
sudo tailscale up

# (Optional) advertise this node as a subnet router so other mesh
# devices can reach your LAN subnets:
sudo tailscale up --advertise-routes=192.168.1.0/24

Enable MagicDNS & HTTPS Certificates

In the Tailscale admin panel (tailscale.com/admin/dns):

  1. Enable MagicDNS — your Pi is now reachable as pi-hostname.tail<xyz>.ts.net from any mesh device.
  2. Enable HTTPS Certificates — Tailscale auto-provisions Let's Encrypt certs for your mesh hostnames.
  3. Optionally configure an Exit Node on one Pi to route all internet traffic through your home network.
Confirmed: Normal ChatGPT / Anthropic / OpenAI traffic does not route through your Pis. The flow is: your device → internet → OpenAI servers. Your Pis only handle traffic you explicitly send to them. Tailscale creates an outbound-initiated, encrypted mesh — no inbound tunnels are opened without your explicit configuration.

Part 2 — Cloudflare Tunnel

Cloudflare Tunnel (cloudflared) creates an outbound-only connection from your Pi to Cloudflare's edge. This exposes your services at a public HTTPS URL without opening any inbound firewall ports or exposing your home IP.

Install cloudflared

# Debian / Raspberry Pi OS
curl -fsSL https://pkg.cloudflare.com/cloudflare-main.gpg \
  | sudo tee /usr/share/keyrings/cloudflare-main.gpg > /dev/null

echo 'deb [signed-by=/usr/share/keyrings/cloudflare-main.gpg] \
  https://pkg.cloudflare.com/cloudflared any main' \
  | sudo tee /etc/apt/sources.list.d/cloudflared.list

sudo apt update && sudo apt install cloudflared

Authenticate & Create Tunnel

# Log in — opens a browser URL
cloudflared tunnel login

# Create a named tunnel
cloudflared tunnel create blackroad-ai

# Note the tunnel UUID printed — you will use it in the config below

Tunnel Configuration

# ~/.cloudflared/config.yml
tunnel: <your-tunnel-uuid>
credentials-file: /home/pi/.cloudflared/<uuid>.json

ingress:
  # Auth / OAuth — Authentik
  - hostname: auth.yourdomain.com
    service: http://localhost:9000

  # LLM inference — Ollama
  - hostname: ollama.yourdomain.com
    service: http://localhost:11434

  # LLM inference — vLLM (OpenAI-compatible API)
  - hostname: api.yourdomain.com
    service: http://localhost:8000

  # Catch-all (required)
  - service: http_status:404

Route DNS & Start Tunnel

# Point DNS (creates CNAME to Cloudflare automatically)
cloudflared tunnel route dns blackroad-ai auth.yourdomain.com
cloudflared tunnel route dns blackroad-ai ollama.yourdomain.com
cloudflared tunnel route dns blackroad-ai api.yourdomain.com

# Run as a system service (starts on boot)
sudo cloudflared service install
sudo systemctl enable --now cloudflared

Part 3 — Nginx Reverse Proxy (Local Routing)

If you run multiple services on the same Pi, Nginx routes requests to the right container based on path or port. This is optional if each service is on its own port and mapped directly in the Cloudflare Tunnel config above. 

# /etc/nginx/sites-available/blackroad-ai
server {
    listen 80;
    server_name api.yourdomain.com;

    # vLLM / Ollama — OpenAI-compatible endpoint
    location / {
        proxy_pass         http://127.0.0.1:8000;
        proxy_set_header   Host $host;
        proxy_set_header   X-Real-IP $remote_addr;
        proxy_set_header   X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header   X-Forwarded-Proto $scheme;

        # Allow streaming responses (for LLM token streaming)
        proxy_buffering    off;
        proxy_read_timeout 300s;
    }
}

server {
    listen 80;
    server_name auth.yourdomain.com;

    location / {
        proxy_pass         http://127.0.0.1:9000;
        proxy_set_header   Host $host;
        proxy_set_header   X-Real-IP $remote_addr;
        proxy_set_header   X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header   X-Forwarded-Proto $scheme;
    }
}
sudo ln -s /etc/nginx/sites-available/blackroad-ai /etc/nginx/sites-enabled/
sudo nginx -t && sudo systemctl reload nginx

Part 4 — Routing AI Clients to Your Infra

Once your tunnel is live, point any OpenAI-compatible client at your own endpoint instead of api.openai.com: 

# Environment variable override (works with most SDKs)
export OPENAI_BASE_URL=https://api.yourdomain.com/v1
export OPENAI_API_KEY=your-local-api-key

# Or in Python
from openai import OpenAI

client = OpenAI(
    base_url="https://api.yourdomain.com/v1",
    api_key="your-local-api-key",   # Set any non-empty string for Ollama
)

response = client.chat.completions.create(
    model="llama3",     # or any model loaded in Ollama / vLLM
    messages=[{"role": "user", "content": "Hello from my own infra!"}],
)
VS Code / Copilot alternatives: Point Continue or Tabby at your self-hosted endpoint for local AI code completions — no Copilot subscription, no Anthropic, no external data egress.
