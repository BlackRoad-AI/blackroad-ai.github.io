OAuth 2.0 on Your Infra

Replace third-party auth providers (GitHub, Google, OpenAI) with a self-hosted OAuth 2.0 server running entirely on your own hardware — your Pis, your mesh, your rules.

Why Self-Hosted OAuth?

Every authentication request that flows through GitHub OAuth, Google OAuth, or similar providers leaks session metadata to those providers. Running your own OAuth 2.0 authorization server means:

  • ▸ Zero token data leaves your network
  • ▸ Full control over token lifetimes, scopes, and revocation
  • ▸ No dependency on external uptime or policy changes
  • ▸ Route all AI service auth through your Tailscale mesh

Recommended Stack: Authentik

Authentik is a self-hosted identity provider with full OAuth 2.0 / OpenID Connect support, a modern UI, and Docker-native deployment. It runs comfortably on a Raspberry Pi 4.

Docker Compose — Single Pi or Server

# authentik-compose.yml
# Place on your Pi, then: docker compose up -d

services:
  postgresql:
    image: postgres:16-alpine
    restart: unless-stopped
    environment:
      POSTGRES_DB: authentik
      POSTGRES_USER: authentik
      POSTGRES_PASSWORD: ${PG_PASS}
    volumes:
      - pg_data:/var/lib/postgresql/data

  redis:
    image: redis:alpine
    restart: unless-stopped

  authentik-server:
    image: ghcr.io/goauthentik/server:latest
    restart: unless-stopped
    command: server
    environment:
      AUTHENTIK_REDIS__HOST: redis
      AUTHENTIK_POSTGRESQL__HOST: postgresql
      AUTHENTIK_POSTGRESQL__USER: authentik
      AUTHENTIK_POSTGRESQL__NAME: authentik
      AUTHENTIK_POSTGRESQL__PASSWORD: ${PG_PASS}
      AUTHENTIK_SECRET_KEY: ${AUTHENTIK_SECRET_KEY}
    ports:
      - "9000:9000"   # HTTP  — fronted by Cloudflare Tunnel
      - "9443:9443"   # HTTPS — optional direct TLS
    volumes:
      - authentik_media:/media
      - authentik_templates:/templates

  authentik-worker:
    image: ghcr.io/goauthentik/server:latest
    restart: unless-stopped
    command: worker
    environment:
      AUTHENTIK_REDIS__HOST: redis
      AUTHENTIK_POSTGRESQL__HOST: postgresql
      AUTHENTIK_POSTGRESQL__USER: authentik
      AUTHENTIK_POSTGRESQL__NAME: authentik
      AUTHENTIK_POSTGRESQL__PASSWORD: ${PG_PASS}
      AUTHENTIK_SECRET_KEY: ${AUTHENTIK_SECRET_KEY}
    volumes:
      - authentik_media:/media

volumes:
  pg_data:
  authentik_media:
  authentik_templates:
Tip: Generate a strong secret key with: openssl rand -base64 36. Store it in a .env file alongside the compose file.

Required .env

PG_PASS=change_me_strong_password
AUTHENTIK_SECRET_KEY=change_me_run_openssl_rand_base64_36

OAuth 2.0 Flow (Self-Hosted)

After deployment, the authentication flow runs entirely inside your mesh:

Your App
→
Authentik
Pi / Server
→
JWT Token
→
Your App
(authorized)

Nothing touches OpenAI, GitHub, or Anthropic infrastructure for authentication.

Step-by-Step Setup

  1. Deploy Authentik using the Docker Compose file above on any Pi or server in your Tailscale mesh. Run docker compose up -d and wait ~60 seconds for initialization.
  2. Create an Application + Provider in the Authentik admin UI (http://<pi-ip>:9000/if/admin/). Choose OAuth 2.0/OpenID Provider as the provider type.
  3. Set Redirect URIs to your application's callback URL. For services behind Cloudflare Tunnel, use your public tunnel URL (e.g., https://auth.yourdomain.com/callback).
  4. Copy Client ID & Secret from the provider settings into your application's environment variables. Never hard-code them in source.
  5. Expose via Cloudflare Tunnel (see the Infrastructure page) so Authentik is reachable at a stable public URL without opening firewall ports.
  6. Test the flow: visit your app, get redirected to Authentik on your Pi, authenticate, and confirm the token comes back to your app — zero external providers involved.

Compatible Providers & Protocols

Authentik supports multiple protocols; choose the one your applications need:

OAuth 2.0 / OIDC

Standard for web apps, SPAs, mobile. Full PKCE support.

SAML 2.0

Enterprise SSO for legacy apps that require SAML assertions.

LDAP

Drop-in directory server for tools like Grafana, Gitea, etc.

SCIM

Automated user provisioning and de-provisioning across services.

Proxy Provider

Forward-auth for services that have no built-in OAuth support.

RADIUS

Network authentication for routers, VPNs, and Wi-Fi access points.

Replacing GitHub / Anthropic / OpenAI Auth

For any BlackRoad AI service that currently delegates authentication to an external provider, swap the environment variables to point at your Authentik instance: 

# Before — external provider
OAUTH_ISSUER=https://github.com
OAUTH_CLIENT_ID=ghp_xxxxxxxxxxxx

# After — your self-hosted Authentik on Tailscale
OAUTH_ISSUER=https://auth.yourdomain.com/application/o/your-app/
OAUTH_CLIENT_ID=<client-id-from-authentik>
OAUTH_CLIENT_SECRET=<client-secret-from-authentik>
Note on Tailscale MagicDNS: If your Pi has Tailscale, Authentik is also reachable at http://<pi-tailscale-hostname>:9000 from any device in your mesh — no tunnel required for internal services.
